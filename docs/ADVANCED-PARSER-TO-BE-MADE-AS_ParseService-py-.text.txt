1. Context-Aware "Magic" Variables

Allow users to use shortcuts that the parser expands based on the current environment. This is incredibly useful for repetitive tasks.



How it works: The parser scans for keywords like $me, $here, or $last.

Example Input: !move $me to $here

The Parser's Job: It replaces $me with the User ID of the person who typed it and $here with the ID of the voice channel they are currently in.

Value: It saves the user from having to copy-paste IDs or mention themselves.



2. Type-Safe "Choice" and "Literal" Constraints

Instead of allowing any string, the parser restricts input to a specific set of words (Enums).



Example Input: !weather --unit kelvin

The Parser's Job: It checks the unit argument against a list of allowed strings: ['celsius', 'fahrenheit'].

Advanced Error: "Error: kelvin is not a supported unit. Please choose from: celsius, fahrenheit."

Technical Implementation: In Python, you would use Literal['celsius', 'fahrenheit'] in your type hints.

1. Multi-Step "Conversational" Parsing
Sometimes a user's input is incomplete. Instead of throwing an error, an advanced parser enters a "pending" state and asks for the missing pieces.

User Input: !ban @user

The Parser's Job: It realizes the reason argument is mandatory but missing.

Action: It pauses the command execution, sends a message saying "What is the reason for this ban?", and "listens" for the next message to fill that specific slot in the parser.

Result: The command finally executes once the "Parser State" is satisfied.




For RPG or Economy bots, users often want to do math. A basic parser sees 10 + 5; an advanced one calculates it.

Input: !give @user 50 * 2 + 25

The Parser's Job: It uses a Math Lexer to ensure the string only contains numbers and operators (avoiding eval() security risks) and passes the result (125) to the function.

Advanced Twist: Support for die notation like !roll 2d20 + 5.


2. Argument Overloading
This allows one command to behave differently based on the data type of the input, all through hardcoded checks.

Logic: !info [Input]

The Parser's Job: 1.  if input.isdigit() → Search for a User ID.
2.  if input.startswith('<@') → Parse as a Mention.
3.  if input.startswith('#') → Search for a Channel.

Result: The user doesn't have to tell the bot what they are looking up; the parser "sniffs" the string pattern and routes it to the correct database query.

3. Escaped Character Handling
Advanced parsers must handle "special" characters like \" or \\. This prevents the parser from breaking when a user wants to include a quote inside a quote.

Example Input: !set_bio "I am \"The Best\" at Python"

The Parser's Job: Instead of splitting at the second quote, the hardcoded logic looks for the backslash \ as an "escape" signal. It ignores the following quote and continues capturing the string until it finds an unescaped ".
